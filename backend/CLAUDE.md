# Backend — Internal Reference

Everything backend developers need. For project overview, see root `CLAUDE.md`. **Keep this file under 200 lines.**

## Layer rules

- `routers/` → `services/` → `models/` only. Services must never import from routers.
- `PromptLoader.load()` for static templates (no variables). `PromptLoader.render()` for templates with `{{variables}}`.
- `AnalysisResult.task_type` is a `Literal`: `coding`, `writing`, `analysis`, `creative`, `data`, `system`, `general`. `selected_strategy` is a plain `str` validated at runtime against `prompts/strategies/`. `intent_label` (3-6 words) and `domain` (resolved via `DomainResolver`) default to `"general"`.

## Pipeline architecture

- **3 phases**: analyze → optimize → score. Each is an independent LLM call with fresh context. Orchestrated by `pipeline.py`
- **Hybrid Phase Routing** (v0.4.2): analyze / score / suggest always run on the internal provider; optimize routes through `MCPSamplingProvider` when the caller is sampling-capable, avoiding the 5-round-trip penalty of the prior sampling-only path. The dedicated `sampling_pipeline.run_sampling_pipeline()` + `run_sampling_analyze()` entry points are retained for the analyze-only diagnostic path and for legacy call sites — `tools/optimize` and `tools/analyze` delegate to them when `tier=sampling`
- **Analyze effort clamp**: `clamp_analyze_effort()` in `pipeline_constants.py` enforces `ANALYZE_EFFORT_CEILING='high'` at all three analyze call sites (`pipeline.py`, `batch_pipeline.py`, `refinement_service.py`). `max`/`xhigh` downshift to `high`; `medium`/`low` pass through. Does NOT apply to optimize/score — they benefit from deep thinking
- **Streaming**: optimize/refine use `messages.stream()` to prevent HTTP timeouts (up to 128K tokens)
- **Enrichment profiles**: auto-selected per request via `select_enrichment_profile()` (pure function). `code_aware` (all layers, coding+repo+history), `knowledge_work` (skip codebase context, non-coding tasks), `cold_start` (skip strategy intelligence + patterns, < 10 optimizations). Profile tracked in `enrichment_meta["enrichment_profile"]`
- **Codebase context**: pre-computed in background on repo link/reindex. Two layers: cached **Sonnet** synthesis (architectural overview in `RepoIndexMeta.explore_synthesis` — long-context reading favours Sonnet; cached per repo/branch/SHA so cost delta is negligible; per-run JSONL trace emitted via `phase="explore_synthesis"`) + per-prompt curated retrieval (full source delivery via `RepoFileIndex.content`, 80K char cap, task-gated: skipped for non-coding prompts, import-graph + doc-ref expansion, source-type balance via `INDEX_CURATED_DOC_CAP_RATIO`, cross-domain filtering, skip-and-continue packing with `_MAX_BUDGET_SKIPS=5`, 5-min TTL). **Repo relevance gate** (B0): `compute_repo_relevance()` returns `(cosine, info_dict)` with single cosine floor `REPO_RELEVANCE_FLOOR=0.15` against a project-anchored synthesis. Anchor embedding prepends `Project: {repo_full_name}\n` before the synthesis and appends up to 500 indexed file paths as `Components:` (fetched lazily from `RepoFileIndex WHERE embedding IS NOT NULL ORDER BY file_path`, stride-sampled at 100 to respect MiniLM's 512-token window). Same-stack-different-project separation rides on project-name signal, not vocabulary overlap. `extract_domain_vocab(synthesis, file_paths)` retained for `domain_matches` info (UI attribution) — never a gate. Reason codes: `above_floor` / `below_floor`. Tracked in `enrichment_meta` as `repo_relevance_score`/`repo_relevance_info`/`repo_relevance_skipped`/`repo_relevance_anchor_paths`. Workspace guidance (manifest scanning) folded in as fallback when synthesis absent (but not when relevance gate fired). Test files excluded from indexing (`_is_test_file()`). Source-type classifier (`_classify_source_type()`: code/docs/config) balances retrieval. Incremental refresh via `REPO_INDEX_REFRESH_INTERVAL` (default 600s, SHA diff, upsert via unique composite index). Heuristic analysis runs for ALL tiers (domain detection). Zero request-time LLM calls except the confidence-gated A4 Haiku fallback (wrapped in `call_provider_with_retry` so transient rate-limit / overload errors retry before degrading to heuristic-only)
- **Strategy intelligence**: unified `resolve_strategy_intelligence()` merges score-based strategy rankings + user feedback affinities + blocked strategies + anti-patterns + domain keywords into single advisory. C1 domain-relaxed fallback: when exact `task_type+domain` returns empty, falls back to `task_type`-only across all domains (separate scope tracking for strategy vs anti-pattern). Gated by `enable_strategy_intelligence` preference
- **Divergence detection**: `detect_divergences()` compares tech mentions in prompt vs codebase context. `_TECH_VOCABULARY` (~50 techs, word-boundary matching). TS/JS compatible, Redis always additive. `EnrichedContext.divergence_alerts` renders 4-category intent classification instructions (OVERSIGHT/DELIBERATE CHANGE/UPGRADE/STANDALONE). `{{divergence_alerts}}` injected between `</codebase-context>` and `<strategy-intelligence>` in optimize/passthrough/refine templates
- **Context delivery**: all tiers receive profile-gated context via `ContextEnrichmentService.enrich()`. 4 context_sources keys: `codebase_context`, `strategy_intelligence`, `applied_patterns`, `heuristic_analysis`. Template render key is `"strategy_intelligence"` everywhere. Passthrough/refine tiers get full `auto_inject_patterns()` in enrichment; internal/sampling defer to pipeline (provenance recording needs `optimization_id`). Few-shot examples in ALL tiers including passthrough. `applied_pattern_texts` stored in `enrichment_meta` for post-optimization UI attribution. `classification_agreement` singleton tracks E1 heuristic vs LLM agreement + strategy intel hit rate
- **Template lifecycle**: `PromptLifecycleService` auto-promotes active→mature (members≥5, coherence≥0.7, score≥7.0). On mature→template thresholds (usage≥3, score≥7.5), `TemplateService.fork_from_cluster()` mints an immutable `PromptTemplate` row; the source cluster stays at `state='mature'`. Warm Phase 0 reconciles `template_count` and auto-retires templates whose source degrades (avg_score<6.0) or is archived (direct query on `state='archived' AND template_count > 0`). Phase 4 recomputes `preferred_strategy` for clusters with `template_count > 0`.
- **Pattern matching**: `match_prompt()` uses raw embeddings (no composite fusion — ensures cross-process consistency between backend and MCP). Family search excludes structural nodes (`EXCLUDED_STRUCTURAL_STATES`). Fallback includes active, candidate, mature, and template states. Leaf clusters load patterns directly when no children. Thresholds: family 0.55, cluster 0.45, candidate 0.65. LRU embedding cache (50 entries, 5-min TTL). Diagnostic logging on no-match (best scores, thresholds, fusion status). Match endpoint rate-limited (30/min) with max_length=8000
- **Scoring**: skippable via `enable_scoring` preference (lean mode = 2 LLM calls). A/B randomized presentation + hybrid scoring
- **Hybrid scoring**: LLM + heuristic blended per dimension (structure 50%, specificity 40%, clarity 40%, conciseness/faithfulness 20%; **technical-prompt conciseness blend bumped to 0.35 in v0.4.7** when high technical-noun density is detected — repeated domain vocabulary should weigh the LLM signal more, not get TTR-penalized as verbose). **Overall weighted mean v4 (v0.4.9, F3)**: per-task-type weights via `get_dimension_weights(task_type)`. Default `DIMENSION_WEIGHTS` (non-analysis): clarity 0.22 / specificity 0.22 / structure 0.15 / faithfulness 0.26 / conciseness 0.15. `ANALYSIS_DIMENSION_WEIGHTS` (audit-class): clarity 0.25 / specificity 0.25 / structure 0.20 / faithfulness 0.20 / conciseness 0.10 — recognizes that audit prompts carry their substance in framing/citations/finding-list structure rather than asserting truth, and are necessarily longer than feature prompts. `DimensionScores` exposes both an arg-free `@property def overall` (uses default weights, ~30 backward-compat call sites) and a sibling `compute_overall(task_type=None)` method (uses per-task-type weights, opt-in for callers with task_type in scope). Module-level invariant assertions in `pipeline_contracts.py` fail-fast on any future weight-sum edit. **F1 specificity backtick category (v0.4.9)**: `heuristic_specificity` 11th category `(r"`[a-zA-Z_][a-zA-Z0-9_./:-]*`", 0, 2.0)` credits backtick-wrapped code identifiers — audit prompts citing real code references earn structural specificity credit (3 hits = +1.6, saturates at +2.0). Z-score normalization when ≥30 samples and `stddev > ZSCORE_MIN_STDDEV`; **F2 (v0.4.9): bumped 0.3 → 0.5** to bypass z-norm on narrow-distribution task types (audits cluster at stddev ~0.35, where z-norm floor-capped legitimately adequate raw scores). **C5 asymmetric cap (v0.4.7)**: floor only at `-2.0`, ceiling uncapped, preserves legitimate above-average upside. Divergence flags at >2.5pt gap. **F5 false-premise flag (v0.4.9)**: when `task_type=='analysis' AND dim=='faithfulness' AND llm_raw < 5.0 AND technical_dense=True`, `divergence_flags` gains `'possible_false_premise'` — surfaces audits whose surface symbol density masks wrong premises. Purely additive, does not perturb scores. Passthrough clamped [1.0, 10.0], excluded from z-score distribution. **C2 length budget guidance (v0.4.7)** in analysis_summary based on `_orig_conc` and `_orig_len` — discourages optimizer over-expansion on prompts that already score well on conciseness
- **Heuristic baseline scores (C4, v0.4.7)**: `Optimization.heuristic_baseline_scores` JSON column stores deterministic `HeuristicScorer.score_prompt(raw_prompt)` directly — distinct from `original_scores` (LLM+heuristic blended, A/B-presentation contaminated). `improvement_score` derivation uses `heuristic_lift` so deltas reflect real optimization gain rather than scoring-method drift. Migration `d3f5a8c91024_heuristic_baseline_pattern_usefulness` adds the column + `useful_count`/`unused_count` counters on `OptimizationPattern`; idempotent via `inspector.get_columns()` guards
- **Pattern usefulness counters (T1.3-lite, v0.4.7)**: `record_pattern_usefulness()` increments `OptimizationPattern.useful_count` when host overall ≥ `PATTERN_USEFUL_FLOOR=7.5` and `unused_count` when ≤ `PATTERN_UNUSED_CEILING=6.5`; feeds future pattern-quality decay + GlobalPattern lifecycle
- **Passthrough**: `prepare_optimization` assembles prompt → external LLM processes → `save_result` persists with heuristic-only scoring

## Routing internals

Process-level singleton `RoutingManager`. `resolve_route()` is a pure function (no I/O):

| Priority | Tier | Condition | Degrade path |
|----------|------|-----------|--------------|
| 1 | `passthrough` | `force_passthrough=True` | none |
| 2 | `sampling` | `force_sampling=True` + MCP caller + sampling capable | → internal → passthrough |
| 3 | `internal` | Provider detected | none |
| 4 | `sampling` | MCP caller + sampling capable (auto) | → passthrough |
| 5 | `passthrough` | Fallback | none |

REST callers never reach sampling tiers — only MCP tool invocations can route to sampling. `_can_sample()` narrowed to `ctx.caller == "mcp"`; auto-path tries internal (tier 3) before auto-sampling (tier 4). `_write_optimistic_session` preserves current `sampling_capable` on session-less reconnects (both VSCode bridge + plain Claude Code reconnect without session ID) — the real initialize handshake sets the final value ms later.

**Singleton pattern**: all singletons guarded by `_process_initialized` in `mcp_server.py`. Lifespan exit has **no cleanup** — singletons survive all sessions. `_clear_stale_session()` runs at process startup only.

**Disconnect signals**: `on_mcp_disconnect()` (all SSE closed → clears both `mcp_connected` + `sampling_capable`) vs `on_sampling_disconnect()` (sampling SSE closed, non-sampling remain → clears only `sampling_capable`). Both persist to `mcp_session.json` and broadcast `routing_state_changed`.

**Disconnect debounce + initialize suppression** (v0.4.7): Claude Code (and similar MCP clients) opens and closes SSE sessions per tool call (every 30–110s on observed traffic). Pre-fix this produced visible status-flicker every cycle plus ~994 cross-process publishes per 30 minutes. Three changes close the loop: (1) `on_mcp_disconnect()` mutates state immediately but defers the broadcast for `DISCONNECT_DEBOUNCE_SECONDS=3.0` via `loop.create_task(_deferred_disconnect_broadcast())`; if `on_mcp_initialize()` fires within the window it cancels the pending task. (2) `on_mcp_disconnect()` snapshots the pre-mutation `sampling_capable` to `_pre_disconnect_sampling`; on debounce-cancel, `on_mcp_initialize()` compares the incoming value against the snapshot and **suppresses the matching initialize broadcast** when capability is unchanged — halves the per-tool-call publish volume. (3) Sustained disconnects past the window propagate normally and clear the snapshot. Falls back to immediate broadcast when no event loop is available (sync test paths). `routing.disconnect_averted` log demoted INFO → DEBUG.

**Recovery trust-fresh-file** (v0.4.7): `_recover_state()` now trusts `sampling_capable` + `mcp_connected` from `mcp_session.json` when BOTH `is_capability_fresh(data)` (written_at within `MCP_CAPABILITY_STALENESS_MINUTES=30`) AND `not detect_disconnect(data)` (active SSE streams or fresh activity on legacy files). Eliminates the ~60s blackhole window when the FastAPI backend restarts (uvicorn auto-reload) while the MCP server process is still up with an authenticated client. If either signal disagrees, falls back to the conservative `wait-for-handshake` behavior.

**Middleware** (`_CapabilityDetectionMiddleware`): intercepts `initialize` JSON-RPC. Dual-layer guard prevents non-sampling clients from overwriting `sampling_capable=True`: primary check on RoutingManager state + secondary on `_sampling_sse_sessions` set. Activity tracking: sampling clients refresh `last_activity`; all clients keep session file fresh.

**Disconnect checker**: 30s poll. Connected mode: check `last_activity` staleness (>60s), read `mcp_session.json` before disconnecting. Disconnected mode: poll file for reconnection.

**Thread-safety**: no `await` between `_state` read and `_update_state()` write. Safe under asyncio cooperative scheduling. See `docs/routing-architecture.md` for diagrams.

## Sampling pipeline internals

End-to-end: MCP tool call → `handle_optimize()` → `routing.resolve()` → `sampling_pipeline.run_sampling_pipeline()` → Phase 1: Analyze → Phase 2: Optimize → Phase 3: Score → Phase 4: Suggest → persist + events. Codebase context arrives pre-computed via `ContextEnrichmentService.enrich()` (no per-request explore LLM call).

**Structured output fallback**: (1) Tool calling via `create_message(tools=..., tool_choice=required)` → parse `tool_use` block. (2) On `McpError`: inject JSON schema as text → `_parse_text_response()` (direct JSON, code block, brace-depth). (3) Analyze-only: `_build_analysis_from_text()` keyword classification.

**Free-text phases**: `OptimizationResult` and `SuggestionsOutput` skip JSON schema to preserve markdown quality. All others get explicit schema.

**Text cleaning** (`app/utils/text_cleanup.py`): `strip_meta_header()` + `split_prompt_and_changes()` (14 marker patterns). Runs before heuristic scoring. Shared by sampling, MCP save_result, REST passthrough. v0.4.5 adds `normalize_sub_domain_label(raw, max_len=30)` — single source of truth for sub-domain label canonicalization (lowercase + space/underscore→hyphen + collapse multi-hyphens + strip leading/trailing + word-boundary truncation); used by both `taxonomy/labeling.py:generate_qualifier_vocabulary` (vocab group names) AND `engine._propose_sub_domains` (sub-domain creation), guaranteeing the format is coherent at both stages.

**Per-phase model capture**: `result.model` from each `create_message()` persisted to DB. IDE selects model freely — no advisory hints sent.

## Taxonomy engine (`services/taxonomy/`)

Process singleton (`get_engine()`/`set_engine()`). **Hierarchy**: project → domain → sub-domain → cluster → optimizations (ADR-005). Three paths: **hot** (per-optimization embed + two-tier project-scoped cosine assignment: in-project first, cross-project fallback +0.15 boost), **warm** (two execution groups: lifecycle [Phases 0–4, dirty-cluster-gated] + maintenance [Phases 5–6, cadence-gated via `MAINTENANCE_CYCLE_INTERVAL=6` + `_maintenance_pending` retry flag]), **cold** (full refit + UMAP 3D + OKLab coloring + Haiku labeling + domain-link restoration + member_count reconciliation + project_ids rebuild). `execute_maintenance_phases()` runs Phase 5 (discover with try/except + retry), Phase 5.5 (sub-domain archival), Phase 6 (audit) independently of dirty clusters. **Dirty-set tracking**: `dict[str, str|None]` (cluster→project). **Adaptive scheduler**: `AdaptiveScheduler` with linear regression boundary; all-dirty vs per-project budget mode (proportional quotas, `_MIN_QUOTA=3` floor, per-project starvation guard at 3 cycles). Warm-path Phase 0 reconciles member_count for both regular clusters and domain nodes; prunes archived clusters >24h old with 0 members. 30s debounce after `taxonomy_changed` SSE events.

**Merge threshold**: adaptive `0.55 + 0.04 * log2(1 + member_count)` — grows with cluster size to prevent centroid-drift mega-clusters. Task_type mismatch penalty (-0.05) for cross-type merges. Used by hot, warm, and cold paths.

**Bookkeeping invariants**: `cluster.member_count == COUNT(optimizations WHERE cluster_id = cluster.id)`. Hot path decrements old cluster on reassignment. Merge zeros loser. Retire increments target. Cold path reconciles from Optimization rows.

**Cold path safety**: Domain nodes excluded from HDBSCAN. Self-referencing parent_ids prevented. Post-HDBSCAN domain-link restoration ensures all active clusters parent to their domain node.

**Quality**: 5-dimension Q_system (coherence, separation, coverage, DBCV, stability) with adaptive weights. Domain floor: coherence ≥0.3.

**Domain discovery**: `_propose_domains()` from coherent "general" sub-populations (≥3 members, ≥0.3 coherence, ≥60% consistent `domain_raw`). Five guardrails: color pinning, retire exemption, merge gate, coherence floor, split→candidates. Ceiling at 30. **Sub-domain discovery**: `_propose_sub_domains()` is fully organic — enriched Haiku-generated vocabulary from cluster labels (`generated_qualifiers` in domain node metadata, cached in `DomainSignalLoader`). `generate_qualifier_vocabulary()` receives per-cluster centroid similarity matrix + intent labels + domain_raw qualifier distribution (`ClusterVocabContext` dataclass, `_VOCAB_SIM_HIGH=0.7`/`_VOCAB_SIM_LOW=0.3`, `None` for unknown cells) **plus (v0.4.7) two new optional kwargs threaded from `engine.py:_propose_sub_domains`**: `domain_signal_keywords: list[tuple[str, float]]` (top TF-IDF orphans with normalized weights) + `existing_vocab_groups: list[str]` (previous group names for continuity). Substring-coverage filter at the rendering layer drops orphans already covered by a current group; the `domain_label` itself is excluded. Closes the organic feedback loop — TF-IDF surfaces a theme → cascade records it → next vocab regen sees it → Haiku decides absorb-or-ignore → next cycle records via source 2 instead of source 3. Post-generation quality metric → `vocab_generated_enriched` event + health `avg_vocab_quality`. **Phase 4.95** (`_propose_sub_domains(vocab_only=True)`) runs in an isolated DB session so vocab staleness doesn't short-circuit Phase 5; `family_ops.merge_meta_pattern` wrapped in SAVEPOINT. Three-source qualifier cascade extracted into shared primitive `compute_qualifier_cascade()` in `sub_domain_readiness.py` — consumed by both discovery and `/api/domains/readiness` for zero drift. Source keys standardized on `tf_idf` (was `dynamic`). **`_extract_domain_keywords()` (v0.4.7)** dual-mode: domain-node input aggregates `Optimization.raw_prompt` across active/mature descendant clusters (domain nodes never own opts directly, so the previous `cluster_id == domain.id` query returned 0 rows and `signal_keywords=[]`); regular-cluster input keeps the direct `cluster_id` query for the seed-cluster path at `_create_domain_node`. Output min-max normalized so the top keyword weighs `1.0` and the K-th preserves the relative ratio — without normalization the cascade's `weight >= 0.5` admit gate filters out every keyword (raw TF-IDF means cluster in 0.05–0.20 range). Adaptive consistency threshold `max(0.40, 0.60 - 0.004 * members)`. No permanent lock — domains with existing sub-domains are re-evaluated; new sub-domains form alongside existing ones. **Sub-domain re-evaluation hardening (v0.4.8, audit `docs/audits/sub-domain-regression-2026-04-27.md` R1-R8)**: `_reevaluate_sub_domains()` dissolves sub-domains when **shrunk** consistency falls below `SUB_DOMAIN_DISSOLUTION_CONSISTENCY_FLOOR=0.25` — Bayesian Beta-Binomial posterior `(matching + α_prior) / (total_opts + K)` with `SUB_DOMAIN_DISSOLUTION_PRIOR_STRENGTH=10` centered at `SUB_DOMAIN_DISSOLUTION_PRIOR_CENTER=0.40` prevents single-prompt small-N noise (R1). Grace gate `SUB_DOMAIN_DISSOLUTION_MIN_AGE_HOURS=24` (was 6, R2). Empty-snapshot guard skips re-eval and emits `sub_domain_reevaluation_skipped` event with `reason=empty_vocab_snapshot` when `cluster_metadata.generated_qualifiers` is empty (R3). Per-opt three-source matching cascade extracted to pure `match_opt_to_sub_domain_vocab() -> SubDomainMatchResult` in `sub_domain_readiness.py` (R4 — single source of truth, no inline-vs-cascade drift). Both `sub_domain_reevaluated` and `sub_domain_dissolved` event contexts gain `shrunk_consistency_pct`, `prior_strength`, `matching_members`, plus up to 3 `sample_match_failures` (`SUB_DOMAIN_FAILURE_SAMPLES=3`, fields truncated to `SUB_DOMAIN_FAILURE_FIELD_TRUNCATE=80` chars; `cluster_id` preserved unsanitized) for forensic reconstruction (R5). Operator recovery via `engine.rebuild_sub_domains()` exposed at `POST /api/domains/{id}/rebuild-sub-domains` (10/min, R6) — Pydantic `ge=0.25` floor + runtime defense, `db.begin_nested()` SAVEPOINT for partial-failure rollback, idempotent on `skipped_existing`, emits `sub_domain_rebuild_invoked` always and publishes `taxonomy_changed` only on non-dry creation. `vocab_generated_enriched` event gains `previous_groups`/`new_groups`/`overlap_pct` (Jaccard %) plus a WARNING log on `<50%` non-bootstrap overlap (R7). Module-level `_validate_threshold_invariants()` in `_constants.py` runs at import time and fails fast if `SUB_DOMAIN_QUALIFIER_CONSISTENCY_LOW <= SUB_DOMAIN_DISSOLUTION_CONSISTENCY_FLOOR` (R8). Dissolution reparents clusters to top-level domain, merges meta-patterns (prompts never lost). `dissolved_this_cycle` set prevents flip-flop. `DomainResolver.remove_label()` clears dissolved labels. Cross-sub-domain merges logged for observability. **Domain lifecycle**: `_reevaluate_domains()` evaluates top-level domains with 5 guards (general permanent, sub-domain anchor, age ≥48h, member ceiling ≤5, consistency <15% Source 1 only). Shared `_dissolve_node()` for both domain and sub-domain dissolution (reparent clusters, merge patterns, archive, clear all 4 indices + resolver + signal loader). Seed domains have no special protection per ADR-006. Phase 5 order: sub-domain reeval → domain reeval → domain discovery → sub-domain discovery → existing post-discovery ops.

**Multi-embedding HDBSCAN**: warm/cold path clustering uses blended embeddings (`blend_embeddings()`: 0.55 raw + 0.20 optimized + 0.15 transformation + 0.10 qualifier, configurable via `_constants.py`). Hot-path assignment stays raw-only (speed + bootstrap avoidance). `Optimization.qualifier_embedding` stores 384-dim qualifier keyword vector from organic vocabulary. Score-weighted centroids: `max(0.1, score / 10.0)`. `TransformationIndex` for technique-space search. `OptimizedEmbeddingIndex` for output-space search. `QualifierIndex` for domain specialization search. Composite fusion (`fusion.py`): 5-signal `PhaseWeights` (topic + transformation + output + pattern + qualifier) with per-phase adaptive weights via `resolve_fused_embedding()`. `PhaseWeights.from_dict()` defaults `w_qualifier=0.0` for backward compat. `compute_score_correlated_target()` skips qualifier dimension for old profiles. Cross-cluster injection: patterns with `global_source_count >= 3` injected across topic boundaries. Output coherence (pairwise cosine of optimized_embeddings) informs split/merge lifecycle. Dual few-shot retrieval: input-similar (cosine >= 0.50) + output-similar (cosine >= 0.40), re-ranked by `max(sim) * score`.

**Adaptive weight learning (3-layer bootstrap)**: Layer 1: `resolve_contextual_weights()` derives per-optimization weight profiles from task-type bias vectors (`_TASK_TYPE_WEIGHT_BIAS`, 7 types). Layer 2: `compute_score_correlated_target()` discovers which profiles correlate with high scores (z-score weighting via `improvement_score` — preferred over `overall_score` — above-median only, min 10 samples) at global and per-cluster level. Layer 3: per-cluster learned weights stored in `cluster_metadata["learned_phase_weights"]`, blended back into Layer 1 at alpha=0.3. Decay toward learned weights at rate=0.01 (< adaptation alpha=0.05, so learning dominates).

**Observability**: `event_logger.py` — `TaxonomyEventLogger` singleton with JSONL persistence (`data/taxonomy_events/decisions-YYYY-MM-DD.jsonl`, 30-day rotation) + ring buffer (500 events) + `taxonomy_activity` SSE bridge. 17 instrumentation points, 12 op types. All `log_decision()` calls wrapped in `try/except RuntimeError: pass`. Initialized in lifespan via `set_event_logger()` in **both** the backend process (`main.py`) and the MCP server process (`mcp_server.py`) — MCP process score events are forwarded to the backend ring buffer via cross-process HTTP POST (`event_notification.py`). Score events include `intent_label` for human-readable display.

**Split protection**: `merge_protected_until` naive-UTC timestamp on split children (60-min window via `SPLIT_MERGE_PROTECTION_MINUTES`). `split_failures` persisted outside speculative transactions via `PhaseResult.split_attempted_ids` + post-rejection session in `_run_speculative_phase()`. 3-strike cooldown with 25% growth reset. `mega_cluster_prevention` gate blocks merges that would create a new mega-cluster, preventing Groundhog Day variant 3 (same-domain merge re-forming an oversized cluster immediately after a split).

**Groundhog Day prevention**: Four-layer defense against split-merge-split loops. Layer 1: `SPLIT_COHERENCE_EXEMPT` (0.38) exempts stable clusters from splitting; `SPLIT_SIBLING_SIMILARITY_CEILING` (0.75) aborts splits when children are too similar to survive independently. Layer 2: domain-level `split_blocked_hashes` ring buffer (20 entries, 6h TTL) tracks failed split content hashes at the domain node level, surviving cluster ID changes; merge-back detection records hashes when recently-protected clusters merge back within `MERGE_BACK_GRACE_MINUTES` (30).

**Spectral split**: `spectral_split()` in `clustering.py` replaces HDBSCAN as primary split algorithm. Tries k=2,3,4 with silhouette gating (rescaled [0,1], gate=0.15, min group size=3). HDBSCAN retained as fallback. Split children created as `state="candidate"`. Warm-path Phase 0.5 (`phase_evaluate_candidates()`) promotes candidates with coherence ≥ `CANDIDATE_COHERENCE_FLOOR` (0.30) or rejects them (reassigns members to nearest active cluster via `_reassign_to_active()`). Candidates excluded from Q_system computation in speculative phases. `_load_active_nodes(exclude_candidates=True)` for Q-gate calls.

**Dissolution**: `phase_retire()` dissolves small incoherent clusters (coherence < `DISSOLVE_COHERENCE_CEILING` 0.30, ≤ `DISSOLVE_MAX_MEMBERS` 5, ≥ `DISSOLVE_MIN_AGE_HOURS` 2h old). Members reassigned via `_reassign_to_active(exclude_cluster_ids={self})` — domain-aware (prefers same-domain targets, updates `opt.domain` on cross-domain reassignment). Q-gated. `retire/dissolved` event. **Forced split** catches the 6-11 member gap: clusters with coherence < `FORCED_SPLIT_COHERENCE_FLOOR` 0.35 and ≥ `FORCED_SPLIT_MIN_MEMBERS` 6 are split via spectral clustering even below `SPLIT_MIN_MEMBERS` (12). **Outlier reconciliation**: Phase 0 ejects cross-domain members (domain != cluster domain, cosine < 0.40 to centroid) to nearest same-domain cluster when a better fit exists (margin 0.10). Capped at 5 per cluster per cycle.

**Performance**: `split_cluster()` uses 3-phase approach — sequential DB queries, parallel `asyncio.gather` on `generate_label()` calls, sequential object creation. Pattern extraction deferred to Phase 4 (Refresh) via `pattern_stale=True`. Phase 4 parallelizes both label generation AND pattern extraction across stale clusters (taxonomy context pre-computed sequentially, LLM calls in parallel — avoids concurrent DB session access).
