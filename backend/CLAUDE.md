## Layer rules

- `routers/` â†’ `services/` â†’ `models/` only. Services must never import from routers.
- `PromptLoader.load()` for static templates (no variables). `PromptLoader.render()` for templates with `{{variables}}`.
- `AnalysisResult.task_type` is a `Literal`: `coding`, `writing`, `analysis`, `creative`, `data`, `system`, `general`. `selected_strategy` is a plain `str` validated at runtime against `prompts/strategies/`. `intent_label` (3-6 words) and `domain` (resolved via `DomainResolver`) default to `"general"`.

## Pipeline architecture

- **3 phases**: analyze â†’ optimize â†’ score. Each is an independent LLM call with fresh context. Orchestrated by `pipeline.py`
- **Streaming**: optimize/refine use `messages.stream()` to prevent HTTP timeouts (up to 128K tokens)
- **Enrichment profiles**: auto-selected per request via `select_enrichment_profile()` (pure function). `code_aware` (all layers, coding+repo+history), `knowledge_work` (skip codebase context, non-coding tasks), `cold_start` (skip strategy intelligence + patterns, < 10 optimizations). Profile tracked in `enrichment_meta["enrichment_profile"]`
- **Codebase context**: pre-computed in background on repo link/reindex. Two layers: cached Haiku synthesis (architectural overview in `RepoIndexMeta.explore_synthesis`) + per-prompt curated retrieval (full source delivery via `RepoFileIndex.content`, 80K char cap, task-gated: skipped for non-coding prompts, import-graph + doc-ref expansion, source-type balance via `INDEX_CURATED_DOC_CAP_RATIO`, cross-domain filtering, skip-and-continue packing with `_MAX_BUDGET_SKIPS=5`, 5-min TTL). **Repo relevance gate** (B0): hybrid `compute_repo_relevance()` returns `(cosine, info_dict)` with two-stage decision: (1) cosine floor at `REPO_RELEVANCE_FLOOR` (0.20) rejects clearly unrelated prompts, (2) domain entity overlap via `extract_domain_vocab()` requires `REPO_DOMAIN_MIN_OVERLAP` (1) repo-specific term in the prompt â€” prevents same-tech-stack-different-project false positives. Tracked in `enrichment_meta` as `repo_relevance_score`/`repo_relevance_info`/`repo_relevance_skipped`. Workspace guidance (manifest scanning) folded in as fallback when synthesis absent (but not when relevance gate fired). Test files excluded from indexing (`_is_test_file()`). Source-type classifier (`_classify_source_type()`: code/docs/config) balances retrieval. Incremental refresh via `REPO_INDEX_REFRESH_INTERVAL` (default 600s, SHA diff, upsert via unique composite index). Heuristic analysis runs for ALL tiers (domain detection). Zero request-time LLM calls
- **Strategy intelligence**: unified `resolve_strategy_intelligence()` merges score-based strategy rankings + user feedback affinities + blocked strategies + anti-patterns + domain keywords into single advisory. C1 domain-relaxed fallback: when exact `task_type+domain` returns empty, falls back to `task_type`-only across all domains (separate scope tracking for strategy vs anti-pattern). Gated by `enable_strategy_intelligence` preference
- **Divergence detection**: `detect_divergences()` compares tech mentions in prompt vs codebase context. `_TECH_VOCABULARY` (~50 techs, word-boundary matching). TS/JS compatible, Redis always additive. `EnrichedContext.divergence_alerts` renders 4-category intent classification instructions (OVERSIGHT/DELIBERATE CHANGE/UPGRADE/STANDALONE). `{{divergence_alerts}}` injected between `</codebase-context>` and `<strategy-intelligence>` in optimize/passthrough/refine templates
- **Context delivery**: all tiers receive profile-gated context via `ContextEnrichmentService.enrich()`. 4 context_sources keys: `codebase_context`, `strategy_intelligence`, `applied_patterns`, `heuristic_analysis`. Template render key is `"strategy_intelligence"` everywhere. Passthrough/refine tiers get full `auto_inject_patterns()` in enrichment; internal/sampling defer to pipeline (provenance recording needs `optimization_id`). Few-shot examples in ALL tiers including passthrough. `applied_pattern_texts` stored in `enrichment_meta` for post-optimization UI attribution. `classification_agreement` singleton tracks E1 heuristic vs LLM agreement + strategy intel hit rate
- **Template lifecycle**: `PromptLifecycleService` auto-promotes activeâ†’mature (membersâ‰Ą5, coherenceâ‰Ą0.7, scoreâ‰Ą7.0). On matureâ†’template thresholds (usageâ‰Ą3, scoreâ‰Ą7.5), `TemplateService.fork_from_cluster()` mints an immutable `PromptTemplate` row; the source cluster stays at `state='mature'`. Warm Phase 0 reconciles `template_count` and auto-retires templates whose source degrades (avg_score<6.0) or is archived (direct query on `state='archived' AND template_count > 0`). Phase 4 recomputes `preferred_strategy` for clusters with `template_count > 0`.
- **Pattern matching**: `match_prompt()` uses raw embeddings (no composite fusion â€” ensures cross-process consistency between backend and MCP). Family search excludes structural nodes (`EXCLUDED_STRUCTURAL_STATES`). Fallback includes active, candidate, mature, and template states. Leaf clusters load patterns directly when no children. Thresholds: family 0.55, cluster 0.45, candidate 0.65. LRU embedding cache (50 entries, 5-min TTL). Diagnostic logging on no-match (best scores, thresholds, fusion status). Match endpoint rate-limited (30/min) with max_length=8000
- **Scoring**: skippable via `enable_scoring` preference (lean mode = 2 LLM calls). A/B randomized presentation + hybrid scoring
- **Hybrid scoring**: LLM + heuristic blended per dimension (structure 50%, specificity 40%, clarity 40%, conciseness/faithfulness 20%). Overall weighted mean v3: clarity 0.22, specificity 0.22, structure 0.15, faithfulness 0.26, conciseness 0.15. `DIMENSION_WEIGHTS` canonical in `pipeline_contracts.py`. Z-score normalization when â‰Ą30 samples. Divergence flags at >2.5pt gap. Passthrough clamped [1.0, 10.0], excluded from z-score distribution
- **Passthrough**: `prepare_optimization` assembles prompt â†’ external LLM processes â†’ `save_result` persists with heuristic-only scoring

## Routing internals

Process-level singleton `RoutingManager`. `resolve_route()` is a pure function (no I/O):

| Priority | Tier | Condition | Degrade path |
|----------|------|-----------|--------------|
| 1 | `passthrough` | `force_passthrough=True` | none |
| 2 | `sampling` | `force_sampling=True` + MCP caller + sampling capable | â†’ internal â†’ passthrough |
| 3 | `internal` | Provider detected | none |
| 4 | `sampling` | MCP caller + sampling capable (auto) | â†’ passthrough |
| 5 | `passthrough` | Fallback | none |

REST callers never reach sampling tiers â€” only MCP tool invocations can route to sampling.

**Singleton pattern**: all singletons guarded by `_process_initialized` in `mcp_server.py`. Lifespan exit has **no cleanup** â€” singletons survive all sessions. `_clear_stale_session()` runs at process startup only.

**Disconnect signals**: `on_mcp_disconnect()` (all SSE closed â†’ clears both `mcp_connected` + `sampling_capable`) vs `on_sampling_disconnect()` (sampling SSE closed, non-sampling remain â†’ clears only `sampling_capable`). Both persist to `mcp_session.json` and broadcast `routing_state_changed`.

**Middleware** (`_CapabilityDetectionMiddleware`): intercepts `initialize` JSON-RPC. Dual-layer guard prevents non-sampling clients from overwriting `sampling_capable=True`: primary check on RoutingManager state + secondary on `_sampling_sse_sessions` set. Activity tracking: sampling clients refresh `last_activity`; all clients keep session file fresh.

**Disconnect checker**: 30s poll. Connected mode: check `last_activity` staleness (>60s), read `mcp_session.json` before disconnecting. Disconnected mode: poll file for reconnection.

**Thread-safety**: no `await` between `_state` read and `_update_state()` write. Safe under asyncio cooperative scheduling. See `docs/routing-architecture.md` for diagrams.

(2) On `McpError`: inject JSON schema as text â†’ `_parse_text_response()` (direct JSON, code block, brace-depth). (3) Analyze-only: `_build_analysis_from_text()` keyword classification.

## Taxonomy engine (`services/taxonomy/`)

Process singleton (`get_engine()`/`set_engine()`). **Hierarchy**: project â†’ domain â†’ sub-domain â†’ cluster â†’ optimizations (ADR-005). Three paths: **hot** (per-optimization embed + two-tier project-scoped cosine assignment: in-project first, cross-project fallback +0.15 boost), **warm** (two execution groups: lifecycle [Phases 0â€“4, dirty-cluster-gated] + maintenance [Phases 5â€“6, cadence-gated via `MAINTENANCE_CYCLE_INTERVAL=6` + `_maintenance_pending` retry flag]), **cold** (full refit + UMAP 3D + OKLab coloring + Haiku labeling + domain-link restoration + member_count reconciliation + project_ids rebuild). `execute_maintenance_phases()` runs Phase 5 (discover with try/except + retry), Phase 5.5 (sub-domain archival), Phase 6 (audit) independently of dirty clusters. **Dirty-set tracking**: `dict[str, str|None]` (clusterâ†’project). **Adaptive scheduler**: `AdaptiveScheduler` with linear regression boundary; all-dirty vs per-project budget mode (proportional quotas, `_MIN_QUOTA=3` floor, per-project starvation guard at 3 cycles). Warm-path Phase 0 reconciles member_count for both regular clusters and domain nodes; prunes archived clusters >24h old with 0 members. 30s debounce after `taxonomy_changed` SSE events.

**Merge threshold**: adaptive `0.55 + 0.04 * log2(1 + member_count)` â€” grows with cluster size to prevent centroid-drift mega-clusters. Task_type mismatch penalty (-0.05) for cross-type merges. Used by hot, warm, and cold paths.

**Bookkeeping invariants**: `cluster.member_count == COUNT(optimizations WHERE cluster_id = cluster.id)`. Hot path decrements old cluster on reassignment. Merge zeros loser. Retire increments target. Cold path reconciles from Optimization rows.

**Cold path safety**: Domain nodes excluded from HDBSCAN. Self-referencing parent_ids prevented. Post-HDBSCAN domain-link restoration ensures all active clusters parent to their domain node.

**Quality**: 5-dimension Q_system (coherence, separation, coverage, DBCV, stability) with adaptive weights. Domain floor: coherence â‰Ą0.3.

**Domain discovery**: `_propose_domains()` from coherent "general" sub-populations (â‰Ą3 members, â‰Ą0.3 coherence, â‰Ą60% consistent `domain_raw`). Five guardrails: color pinning, retire exemption, merge gate, coherence floor, splitâ†’candidates. Ceiling at 30. **Sub-domain discovery**: `_propose_sub_domains()` is fully organic â€” enriched Haiku-generated vocabulary from cluster labels (`generated_qualifiers` in domain node metadata, cached in `DomainSignalLoader`). `generate_qualifier_vocabulary()` receives per-cluster centroid similarity matrix + intent labels + domain_raw qualifier distribution (`ClusterVocabContext` dataclass, `_VOCAB_SIM_HIGH=0.7`/`_VOCAB_SIM_LOW=0.3`, `None` for unknown cells). Post-generation quality metric â†’ `vocab_generated_enriched` event + health `avg_vocab_quality`. **Phase 4.95** (`_propose_sub_domains(vocab_only=True)`) runs in an isolated DB session so vocab staleness doesn't short-circuit Phase 5; `family_ops.merge_meta_pattern` wrapped in SAVEPOINT. Three-source qualifier cascade extracted into shared primitive `compute_qualifier_cascade()` in `sub_domain_readiness.py` â€” consumed by both discovery and `/api/domains/readiness` for zero drift. Source keys standardized on `tf_idf` (was `dynamic`). Adaptive consistency threshold `max(0.40, 0.60 - 0.004 * members)`. No permanent lock â€” domains with existing sub-domains are re-evaluated; new sub-domains form alongside existing ones. `_reevaluate_sub_domains()` dissolves sub-domains with consistency < `SUB_DOMAIN_DISSOLUTION_CONSISTENCY_FLOOR` (0.25) â€” reparents clusters to top-level domain, merges meta-patterns (prompts never lost). `dissolved_this_cycle` set prevents flip-flop. `DomainResolver.remove_label()` clears dissolved labels. Cross-sub-domain merges logged for observability. **Domain lifecycle**: `_reevaluate_domains()` evaluates top-level domains with 5 guards (general permanent, sub-domain anchor, age â‰Ą48h, member ceiling â‰¤5, consistency <15% Source 1 only). Shared `_dissolve_node()` for both domain and sub-domain dissolution (reparent clusters, merge patterns, archive, clear all 4 indices + resolver + signal loader). Seed domains have no special protection per ADR-006. Phase 5 order: sub-domain reeval â†’ domain reeval â†’ domain discovery â†’ sub-domain discovery â†’ existing post-discovery ops.

**Multi-embedding HDBSCAN**: warm/cold path clustering uses blended embeddings (`blend_embeddings()`: 0.55 raw + 0.20 optimized + 0.15 transformation + 0.10 qualifier, configurable via `_constants.py`). Hot-path assignment stays raw-only (speed + bootstrap avoidance). `Optimization.qualifier_embedding` stores 384-dim qualifier keyword vector from organic vocabulary. Score-weighted centroids: `max(0.1, score / 10.0)`. `TransformationIndex` for technique-space search. `OptimizedEmbeddingIndex` for output-space search. `QualifierIndex` for domain specialization search. Composite fusion (`fusion.py`): 5-signal `PhaseWeights` (topic + transformation + output + pattern + qualifier) with per-phase adaptive weights via `resolve_fused_embedding()`. `PhaseWeights.from_dict()` defaults `w_qualifier=0.0` for backward compat. `compute_score_correlated_target()` skips qualifier dimension for old profiles. Cross-cluster injection: patterns with `global_source_count >= 3` injected across topic boundaries. Output coherence (pairwise cosine of optimized_embeddings) informs split/merge lifecycle. Dual few-shot retrieval: input-similar (cosine >= 0.50) + output-similar (cosine >= 0.40), re-ranked by `max(sim) * score`.

**Adaptive weight learning (3-layer bootstrap)**: Layer 1: `resolve_contextual_weights()` derives per-optimization weight profiles from task-type bias vectors (`_TASK_TYPE_WEIGHT_BIAS`, 7 types). Layer 2: `compute_score_correlated_target()` discovers which profiles correlate with high scores (z-score weighting via `improvement_score` â€” preferred over `overall_score` â€” above-median only, min 10 samples) at global and per-cluster level. Layer 3: per-cluster learned weights stored in `cluster_metadata["learned_phase_weights"]`, blended back into Layer 1 at alpha=0.3. Decay toward learned weights at rate=0.01 (< adaptation alpha=0.05, so learning dominates).

**Observability**: `event_logger.py` â€” `TaxonomyEventLogger` singleton with JSONL persistence (`data/taxonomy_events/decisions-YYYY-MM-DD.jsonl`, 30-day rotation) + ring buffer (500 events) + `taxonomy_activity` SSE bridge. 17 instrumentation points, 12 op types. **Split protection**: `merge_protected_until` naive-UTC timestamp on split children (60-min window via `SPLIT_MERGE_PROTECTION_MINUTES`). `split_failures` persisted outside speculative transactions via `PhaseResult.split_attempted_ids` + post-rejection session in `_run_speculative_phase()`. 3-strike cooldown with 25% growth reset. `mega_cluster_prevention` gate blocks merges that would create a new mega-cluster, preventing Groundhog Day variant 3 (same-domain merge re-forming an oversized cluster immediately after a split).

**Groundhog Day prevention**: Four-layer defense against split-merge-split loops. Layer 1: `SPLIT_COHERENCE_EXEMPT` (0.38) exempts stable clusters from splitting; `SPLIT_SIBLING_SIMILARITY_CEILING` (0.75) aborts splits when children are too similar to survive independently. Layer 2: domain-level `split_blocked_hashes` ring buffer (20 entries, 6h TTL) tracks failed split content hashes at the domain node level, surviving cluster ID changes; merge-back detection records hashes when recently-protected clusters merge back within `MERGE_BACK_GRACE_MINUTES` (30).

**Spectral split**: `spectral_split()` in `clustering.py` replaces HDBSCAN as primary split algorithm. Tries k=2,3,4 with silhouette gating (rescaled [0,1], gate=0.15, min group size=3). HDBSCAN retained as fallback. Split children created as `state="candidate"`. Warm-path Phase 0.5 (`phase_evaluate_candidates()`) promotes candidates with coherence â‰Ą `CANDIDATE_COHERENCE_FLOOR` (0.30) or rejects them (reassigns members to nearest active cluster via `_reassign_to_active()`). Candidates excluded from Q_system computation in speculative phases. `_load_active_nodes(exclude_candidates=True)` for Q-gate calls.

**Dissolution**: `phase_retire()` dissolves small incoherent clusters (coherence < `DISSOLVE_COHERENCE_CEILING` 0.30, â‰¤ `DISSOLVE_MAX_MEMBERS` 5, â‰Ą `DISSOLVE_MIN_AGE_HOURS` 2h old). Members reassigned via `_reassign_to_active(exclude_cluster_ids={self})` â€” domain-aware (prefers same-domain targets, updates `opt.domain` on cross-domain reassignment). Q-gated. `retire/dissolved` event. **Forced split** catches the 6-11 member gap: clusters with coherence < `FORCED_SPLIT_COHERENCE_FLOOR` 0.35 and â‰Ą `FORCED_SPLIT_MIN_MEMBERS` 6 are split via spectral clustering even below `SPLIT_MIN_MEMBERS` (12). **Outlier reconciliation**: Phase 0 ejects cross-domain members (domain != cluster domain, cosine < 0.40 to centroid) to nearest same-domain cluster when a better fit exists (margin 0.10). Capped at 5 per cluster per cycle.

**Performance**: `split_cluster()` uses 3-phase approach â€” sequential DB queries, parallel `asyncio.gather` on `generate_label()` calls, sequential object creation. Pattern extraction deferred to Phase 4 (Refresh) via `pattern_stale=True`. Phase 4 parallelizes both label generation AND pattern extraction across stale clusters (taxonomy context pre-computed sequentially, LLM calls in parallel â€” avoids concurrent DB session access).
