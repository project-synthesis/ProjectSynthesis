# Support

## How to get help

| Channel | Best for |
|---------|----------|
| [GitHub Issues](../../issues) | Bug reports, feature requests |
| [GitHub Discussions](../../discussions) | Questions, ideas, show-and-tell |
| [SECURITY.md](../SECURITY.md) | Vulnerability disclosures (private) |

## Before opening an issue

1. Check the [existing issues](../../issues) — your question may already be answered
2. Read [CLAUDE.md](../CLAUDE.md) for architecture and configuration details
3. Check `data/backend.log` and `data/frontend.log` for error context

## Providing a good bug report

Include:
- OS and Python / Node version
- Steps to reproduce
- Expected vs actual behaviour
- Relevant log output (redact any tokens or credentials)

## Terms of Use

Project Synthesis is free for personal and organizational use with no paid tier. There is no SLA or vendor relationship — support is community best-effort. See [TERMS.md](TERMS.md) for the full usage policy (in this directory).

## Out of scope for issues

- General prompt engineering questions (use GitHub Discussions instead)
- Feature requests without a clear use case
- Support for self-hosted deployments with custom modifications
